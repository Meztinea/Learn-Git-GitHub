- Cada vez que se ejecuta commit, se está generando una nueva versión del programa

* GIT tiene tres áreas de trabajo

1. Área de trabajo : Archivos que estámos trabajando, Git no conoce y tampoco sigue los cambios de nuestros
    archivos, los  cambios realizados en esta área pueden perderse pues git no tiene conocimiento de estos    
    archivos ni de sus cambios. 

2. Área de staging : Git puede identificar que estos archivos están sufriendo modificaciones, pero en este
    punto los cambios aún no forman parte oficialmente del historial de versiones del proyecto

3. Área de repositorio : Git identifica los archivos que son parte del proyecto y ha almacenado una versión
    de éste. En este punto los cambios están guardados como una nueva versión



* git add <nombre>
Mueve los archivos del área de trabajo al área de staging
Agrega el archivo al proyecto, indica que este archivo debe ser seguido, pues es parte del proyecto

* git commit 
Mueve los archivos del área de staging al área de repositorio, genera una nueva versión cada vez que es 
    ejecutado

* git status 
Permite conocer el estado de los archivos, indicará cuando:
    * Un archivo no esté siendo seguido
    * Algún archivo ha sufrido cambios

* git commit -m 'my first commit' 
Entre comillas se pueden especificar los cambios realizados en esta nueva versión 

* git log 
Muestra el historial de commits del proyecto, indicando el usuario, correo, fecha e identificador del commit
